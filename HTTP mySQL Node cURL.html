Relación entre métodos HTTP, MySQL y Node.js

En el desarrollo web, HTTP, Node.js y MySQL trabajan juntos para crear aplicaciones dinámicas. Los métodos HTTP permiten que el navegador o un cliente envíe solicitudes al servidor; Node.js recibe esas solicitudes, procesa la lógica de la aplicación y, cuando es necesario, se conecta a MySQL para guardar, consultar, actualizar o eliminar datos.

En otras palabras, HTTP es el medio de comunicación, Node.js es el servidor que interpreta la petición, y MySQL es la base de datos donde se almacenan los datos.

¿Qué son los métodos HTTP?

Los métodos HTTP son instrucciones que indican qué acción quiere realizar el cliente sobre un recurso. Los más usados en aplicaciones web son:

  • GET: obtiene información del servidor.
  • POST: envía datos nuevos al servidor.
  • PUT: actualiza completamente un registro existente.
  • PATCH: actualiza solo una parte de un registro.
  • DELETE: elimina un recurso o registro.

Estos métodos suelen relacionarse con operaciones en una base de datos: por ejemplo, GET normalmente consulta datos en MySQL, POST inserta nuevos registros, PUT o PATCH modifican registros, y DELETE elimina información.

Relación entre HTTP y MySQL con Node.js

Node.js actúa como intermediario entre el cliente y MySQL. Cuando el usuario hace una petición HTTP, Node.js recibe esa solicitud, la interpreta y ejecuta una operación en la base de datos. Después, devuelve una respuesta HTTP con el resultado.

Ejemplo del flujo:

  1. El cliente envía una solicitud GET para consultar usuarios.
  2. Node.js recibe la petición usando, por ejemplo, Express.
  3. Node.js ejecuta una consulta SQL en MySQL.
  4. MySQL devuelve los datos.
  5. Node.js responde al cliente con JSON o HTML.

Gracias a esta relación, es posible crear sistemas como registros de usuarios, tiendas en línea, blogs, paneles de administración y cualquier aplicación que necesite guardar información.

Relación entre métodos HTTP y operaciones en MySQL

Método HTTP Acción en la aplicación Relación con MySQL
GET Obtener datos Realiza una consulta SELECT
POST Crear un nuevo registro Realiza una consulta INSERT
PUT Actualizar un registro completo Realiza una consulta UPDATE
PATCH Actualizar una parte del registro Realiza una consulta UPDATE parcial
DELETE Eliminar datos Realiza una consulta DELETE

Ejemplo con Node.js

En Node.js, normalmente se usa un framework como Express y un conector como mysql2 para comunicar la aplicación con la base de datos MySQL. Cada ruta puede estar asociada a un método HTTP diferente. 

#Consultar
app.get('/usuarios', obtenerUsuarios);

#Insertar
app.post('/usuarios', crearUsuario);

#Actualizar
app.put('/usuarios/:id', actualizarUsuario);

#Eliminar
app.delete('/usuarios/:id', eliminarUsuario);

Así, el servidor decide qué consulta SQL ejecutar según el método HTTP recibido.

¿Qué es cURL?

cURL es una herramienta de línea de comandos que permite enviar solicitudes a un servidor web desde la terminal. Se usa mucho para probar APIs, verificar respuestas y simular peticiones HTTP sin necesidad de usar un navegador.

Con cURL puedes hacer peticiones GET, POST, PUT y DELETE, enviar encabezados, autenticación y datos en formato JSON.

Ejemplos de cURL

# Obtener datos
curl http://localhost:3000/usuarios

# Enviar datos con POST
curl -X POST http://localhost:3000/usuarios -H "Content-Type: application/json" -d '{"nombre":"Ana","edad":20}'

# Actualizar datos con PUT
curl -X PUT http://localhost:3000/usuarios/1 -H "Content-Type: application/json" -d '{"nombre":"Ana López","edad":21}'

# Eliminar datos
curl -X DELETE http://localhost:3000/usuarios/1

cURL es útil para programadores porque permite probar rápidamente si una API funciona correctamente.
